DX Node API Specifications

B2B Peer-to-Peer Data Governance & Transfer Protocol.

Security Baseline

Base API Endpoint

/api/client/dx

All requests must use the POST method with a valid _func identifier in the body.

Mandatory Headers
Key: X-API-Key
Secret: X-API-Secret

1. Discovery & Inspection

Discovery

list_contracts

RBAC Filtered

Retrieves the registry of Data Contracts authorized for your specific Application ID. Includes peer identities and flow directions.

{ "_func": "list_contracts" }
Inspection

get_contract_inspector

Metadata Sync

Exposes the Level 1 (Structural) schema and Level 2 (PII) anonymization rules for a specific contract. Use this to ensure your local payload matches peer requirements.

{ "_func": "get_contract_inspector", "contract_id": "c7b3d8e0-f2a1-4b3c-8d9e-0f1a2b3c4d5e" }

2. Transfer Orchestration

Execution

initiate_transfer

Pushes a Base64-encoded payload to the node. The node automatically enforces the governing Data Contract, performs PII anonymization, and initiates the mTLS handshake with the partner.

3. Forensic Transparency

Governance Audit

view_forensic_payload

The TSI DX Node implements Forensic Mirroring. This function returns the Bit-Perfect version of the data *after* it was processed by the node's anonymization engine. Use this to prove compliance with digital sharing agreements.

{ "_func": "view_forensic_payload", "transfer_id": "8cec1cdd-a15c-49d2-9b9b-8902b20237fb" }
